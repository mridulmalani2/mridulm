# Memory

## Purpose

To retain high-signal insights, patterns, and learnings across analyses.

---

## What to Store

### 1. Recurring Patterns
- Deal structures
- Value creation strategies
- Sector trends

---

### 2. Key Insights
- Non-obvious observations
- Lessons from specific deals

---

### 3. Mistakes
- Incorrect assumptions
- Gaps in analysis

---

### 4. Framework Improvements
- Changes to templates
- Better structuring approaches

---

## Storage Rules

- Only store high-value insights
- Avoid redundancy
- Keep entries concise

---

## Usage

- Refer to past insights when relevant
- Do not force references
- Use memory to improve future outputs
